Create new file `src/lib/xpFormulas.ts` with the following pure functions:

**1. `calculateWeeklyBaseXP(workoutCount: number): number`**

Non-linear scaling per CONTEXT.md and XP-01 requirement. The inflection point is at 3 workouts (minimum target):
- 0 workouts → 0 XP
- 1 workout → 5 XP
- 2 workouts → 15 XP
- 3 workouts → 30 XP (inflection — this is the minimum target)
- 4 workouts → 50 XP (ideal target)
- 5 workouts → 80 XP
- 6+ workouts → 100 XP (god mode reward)

Implementation: Simple if/else chain (not a formula). Clamp input to 0-7 range. Return 0 for negative or invalid input.

**2. `calculateStreakMultiplier(streakWeeks: number): number`**

Tiered multiplier per CONTEXT.md decisions and XP-03 requirement:
- 0-3 weeks → 1.0x (no bonus)
- 4-11 weeks → 1.5x (1 month streak)
- 12-25 weeks → 1.75x (3 month streak)
- 26-51 weeks → 2.0x (6 month streak)
- 52+ weeks → 2.5x (1 year streak, CAPPED — does not grow further)

Implementation: Simple if/else chain descending from highest tier. Return 1.0 for 0 or negative input.

**3. `calculateWeeklyXP(workoutCount: number, streakWeeks: number, weekStatus: 'normal' | 'sick' | 'vacation'): number`**

Combines base XP and streak multiplier:
- FIRST check: if `weekStatus !== 'normal'`, return 0 immediately (per CONTEXT.md: sick/vacation earn 0 XP)
- Calculate: `baseXP * streakMultiplier`
- Apply `Math.round()` to result (per RESEARCH.md pitfall #3)
- Note: Achievement bonus is NOT handled here (flat +100 XP added separately in Phase 5)

**4. `getWeeklyXPBreakdown(workoutCount: number, streakWeeks: number, weekStatus: 'normal' | 'sick' | 'vacation'): WeeklyXPBreakdown`**

Returns decomposed XP values for UI tooltip (Phase 6):
- If `weekStatus !== 'normal'`: return `{ baseXP: 0, streakMultiplier: 1.0, totalXP: 0 }`
- Otherwise: return `{ baseXP, streakMultiplier, totalXP: Math.round(baseXP * streakMultiplier) }`
- `achievementBonus` field intentionally omitted (added in Phase 5)

Import `WeeklyXPBreakdown` from `'../types'` using `import type`.

Follow project conventions:
- Import order: type imports at end (per CLAUDE.md)
- JSDoc comments on all exported functions describing purpose and parameters
- All functions MUST be synchronous (no async, no Promises) — called on every workout toggle
- All functions MUST be pure (no side effects, no state mutation)
- Use `Math.round()` on all final XP values to prevent floating-point display issues
- Pattern after `src/lib/weekUtils.ts` (pure utility functions)
