Create new file `src/hooks/useXP.ts` implementing the XP state management hook.

**Import structure (follow project import order):**
```typescript
import { useState, useEffect, useMemo, useCallback } from 'react';
import { doc, getDoc, setDoc } from 'firebase/firestore';

import { db } from '../lib/firebase';
import { useAuth } from '../contexts/AuthContext';
import { calculateWeeklyXP } from '../lib/xpFormulas';
import { getRankForXP, getNextRank, getXPToNextRank, checkRankUp } from '../lib/ranks';

import type { Rank, LevelUpEvent } from '../types';
import type { WeekRecord } from './useAllWeeks';
```

**Hook signature:**
```typescript
export function useXP(
  weeks: WeekRecord[],
  currentStreak: number,
  unlockedAchievementCount: number,
  weeksLoading: boolean
)
```

Parameters:
- `weeks` — from `useAllWeeks().weeks` (all historical week records)
- `currentStreak` — from `useAllWeeks().stats.currentStreak` (current streak length)
- `unlockedAchievementCount` — from achievement context (number of unlocked achievements for retroactive calc)
- `weeksLoading` — from `useAllWeeks().loading` (prevents premature retroactive calc)

**Internal state:**
- `totalXP: number` (default 0)
- `achievementXP: number` (default 0)
- `loading: boolean` (default true)
- `levelUpEvent: LevelUpEvent | null` (default null)
- `xpLoaded: boolean` (default false) — tracks if initial Firestore load is complete

**Effect 1: Load XP from Firestore on mount (depends on `user`)**
- If no `user`, set `loading = false` and return (guest mode, no XP)
- Read `users/{uid}/stats/current` document via `getDoc`
- If document exists AND `totalXP` field is defined (not undefined):
  - Set `totalXP` from doc data
  - Set `achievementXP` from doc data (default 0)
  - Set `xpLoaded = true`
  - Set `loading = false`
- If document does NOT exist or `totalXP` is undefined:
  - Set `xpLoaded = false` (will trigger retroactive calc in Effect 2)
  - Set `loading = false` for now (retroactive calc will manage its own state)
- On error: `console.error`, set `loading = false`

**Effect 2: Retroactive XP calculation (depends on `user`, `xpLoaded`, `weeks`, `weeksLoading`, `unlockedAchievementCount`)**
- Guard conditions (return early if ANY are true):
  - `!user` (guest)
  - `xpLoaded` (XP already loaded from Firestore — trust it, no recalc)
  - `weeksLoading` (weeks haven't loaded yet)
  - `weeks.length === 0 && !weeksLoading` (no workout history and loading is done — brand new user, skip)
- This triggers when: user exists, XP data was NOT found in Firestore, and weeks have finished loading
- Calculate total workout XP: iterate all weeks, sum `calculateWeeklyXP(week.workoutCount, currentStreak, week.status)` for each
  - Per CONTEXT.md/RESEARCH.md: use current streak for ALL historical weeks (simplified, more rewarding)
- Calculate achievement XP: `unlockedAchievementCount * 100`
- Total XP: workout XP + achievement XP
- Determine rank: `getRankForXP(totalXP)`
- Try to persist to Firestore (`setDoc` with `{ merge: true }`):
  - Write `totalXP`, `currentRankId: rank.id`, `achievementXP` to `stats/current`
  - On success: update local state (`totalXP`, `achievementXP`), set `xpLoaded = true`
  - On error: `console.error('Error calculating retroactive XP:', error)` — do NOT persist partial results, do NOT set `xpLoaded = true` (retry on next load per CONTEXT.md)
- **NO level-up toasts during retroactive** (per RANK-03): don't call `setLevelUpEvent` here
- Log to console: `Retroactive XP granted: ${totalXP} XP (Rank: ${rank.name})`

**Function: `addXP(amount: number, isSilent = false): Promise<void>`**
Wrapped in `useCallback` with deps `[user, totalXP]`:
- If `!user`, return early
- Calculate `newTotalXP = totalXP + amount`
- Check for rank-up: `checkRankUp(totalXP, newTotalXP)`
- If rank-up occurred AND `!isSilent`: `setLevelUpEvent(rankUpEvent)`
- Update local state: `setTotalXP(newTotalXP)`
- Calculate new rank: `getRankForXP(newTotalXP)`
- Persist to Firestore with `setDoc` merge:
  - `totalXP: newTotalXP`, `currentRankId: newRank.id`
  - If `amount > 0` and this is achievement XP, also update `achievementXP` (caller handles this separately)
- On error: `console.error('Error updating XP:', error)`

**Function: `recalculateXP(): Promise<void>`**
Wrapped in `useCallback` with deps `[user, weeks, currentStreak, unlockedAchievementCount]`:
- If `!user`, return early
- Full recalculation from all weeks (same logic as retroactive calc)
- Persist to Firestore
- Update local state
- This is the "nuclear option" for when delta-based recalc might be inaccurate (e.g., after status change on past week)
- Silent mode (no level-up toasts)

**Function: `dismissLevelUp(): void`**
- `setLevelUpEvent(null)`

**Derived values (useMemo with PRIMITIVE dependencies per CONTEXT.md):**
- `currentRank: Rank` = `useMemo(() => getRankForXP(totalXP), [totalXP])`
- `nextRank: Rank | null` = `useMemo(() => getNextRank(currentRank.id), [currentRank.id])`
- `xpToNextRank: number` = `useMemo(() => nextRank ? nextRank.xpThreshold - totalXP : 0, [nextRank, totalXP])`

**CRITICAL: Use PRIMITIVE dependencies in useMemo:**
- `[totalXP]` not `[xpData]`
- `[currentRank.id]` not `[currentRank]`
- This prevents infinite render loops (per RESEARCH.md Pitfall 1)

**Return value:**
```typescript
return {
  totalXP,
  achievementXP,
  currentRank,
  nextRank,
  xpToNextRank,
  loading,
  levelUpEvent,
  addXP,
  recalculateXP,
  dismissLevelUp,
};
```

**Follow project conventions:**
- Functional component hook pattern (like `useStats.ts`, `useAllWeeks.ts`)
- JSDoc comments on the hook and all public functions
- Try-catch around all Firestore operations
- `console.error` for error logging
- Pure TypeScript, no React components
- Import order: React → Firebase → internal libs → types
