```typescript
interface RankShowcaseProps {
  currentRank: Rank;
  totalXP: number;
  xpToNextRank: number;
  nextRank: Rank | null;
}
```

**Component structure (per user decisions):**
1. Outer container: `doom-panel p-3` with centered header
2. Header: title "RANK PROGRESSION" in doom-gold, subtitle below it (Claude's discretion on subtitle text, e.g., "CLIMB THE LADDER")
3. Card list: `space-y-2 mt-3` container, map over `RANKS` array (imported from `src/lib/ranks.ts`)
4. Ascending order: #1 Private at top, #15 Doom Slayer at bottom (RANKS array is already in this order)

**Per-card logic:**
- Determine `isEarned = rank.id <= currentRank.id` and `isCurrent = rank.id === currentRank.id`
- Card classes: `achievement-card p-3 rounded` base
  - Current rank: add `god-mode-glow border-doom-gold` classes + `style={{ willChange: 'box-shadow' }}` for GPU perf
  - Locked/unearned: add `locked` class (applies existing opacity 0.5 + grayscale 0.8)
  - Earned (not current): no extra classes (full opacity, normal appearance)

**Card content layout (flex row, justify-between):**
- Left side (flex-1):
  - Line 1: `#{rank.id} {rank.name}` -- text-[10px] font-bold, color is `rank.color` if earned, `text-gray-600` if locked
  - Line 2: `{rank.tagline}` -- text-[8px] text-gray-400
  - Line 3 (current rank only): progress indicator
    - If nextRank exists: `+{xpToNextRank} XP to {nextRank.name}` in text-[7px] text-doom-gold
    - If no nextRank (max rank): `MAX RANK ACHIEVED` in text-[7px] text-doom-gold
- Right side: `{rank.xpThreshold} XP` -- text-[9px] text-gray-500

**Important constraints (from user decisions):**
- XP threshold as raw number, NO comma formatting (e.g., "5000 XP" not "5,000 XP")
- Reuse .achievement-card styling pattern (do NOT create new CSS classes)
- Reuse .god-mode-glow for current rank (do NOT create new animations)
- Only ONE element gets .god-mode-glow (the current rank card)

**Tailwind color classes note:** The rank color classes (text-gray-400, text-green-400, text-blue-400, etc.) are defined as string literals in src/lib/ranks.ts which is scanned by Tailwind's content config. No safelist needed -- they will be preserved in production builds.
