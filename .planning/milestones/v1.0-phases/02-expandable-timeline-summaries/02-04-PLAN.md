# Plan 02-04: Dashboard Integration & Polish

## Goal

Integrate timeline components into Dashboard below existing 12-week summary, add CSS transitions for smooth animations, and ensure existing Dashboard features remain unchanged. This completes the phase by making the timeline visible to users.

## Context

Plans 02-01 through 02-03b created data layer and UI components. Now we integrate into Dashboard: add timeline below existing sections, wire up useTimelineData hook, implement CSS max-height transitions, and verify existing 12-week summary and day heatmap remain untouched.

**User requirement:** "Timeline appears below existing 12-week summary - preserves familiar flow."

**User decision on current week:** "Current week always visible at top, pinned above timeline sections for quick tracking (separate from historical data)". The existing 12-week grid at the top of Dashboard already satisfies this - it shows the current week plus 11 previous weeks, always visible. The new timeline section will appear below this, providing historical context beyond the 12-week window.

               yearWeeks={yearWeeks}

             />

       </div>
