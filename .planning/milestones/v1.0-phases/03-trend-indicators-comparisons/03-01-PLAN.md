Create trendUtils.ts with percentage change calculation logic handling all edge cases per RESEARCH.md code examples (lines 342-406).

Implementation requirements:
- **TrendData interface**: direction ('up' | 'down' | 'stable'), percentage (number | 'infinity'), display (string ready for UI)
- **calculateTrend function**:
  - Calculate percentage change: ((current - previous) / previous) * 100
  - Round to whole numbers using Math.round() (not toFixed - returns string)
  - Handle division by zero: if previous === 0 and current > 0, return { direction: 'up', percentage: 'infinity', display: '↑ ∞' }
  - Handle both zero: if previous === 0 and current === 0, return { direction: 'stable', percentage: 0, display: '→ 0%' }
  - Handle no previous period: if previous is null/undefined, return null
  - Format display string: arrow (↑ ↓ →) + sign (+ for positive, - auto for negative) + percentage + %
  - Use Unicode arrows: U+2191 ↑ (up), U+2193 ↓ (down), U+2192 → (stable)
  - Arrow direction: up if rounded > 0, down if rounded < 0, stable if rounded === 0

Add comprehensive JSDoc comments explaining edge case handling (division by zero, both zero, no previous period).
