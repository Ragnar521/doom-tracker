Modify `src/hooks/useWeek.ts` to support XP delta calculation when workouts are toggled or week status changes.

**Approach:** Rather than making useWeek depend on useXP (which would create circular dependency issues), add callback parameters that the consuming component can wire to useXP. This keeps useWeek independent and testable.

**1. Add import for calculateWeeklyXP:**
```typescript
import { calculateWeeklyXP } from '../lib/xpFormulas';
```

**2. Add optional callback parameters to useWeek:**
Change the hook signature to accept an options object:
```typescript
interface UseWeekOptions {
  onXPDelta?: (delta: number) => void;
  onXPRecalculate?: () => void;
  currentStreak?: number;
}

export function useWeek(weekId: string = getCurrentWeekId(), options: UseWeekOptions = {})
```

The `onXPDelta` callback is called with the XP difference when a workout is toggled. The `onXPRecalculate` callback is called when week status changes (since status change requires full week XP recalc, not just delta). `currentStreak` is needed for XP formula calculation.

**3. Modify `toggleDay` function:**
After the existing workout toggle logic (before `await saveData(newData)`), add XP delta calculation:

```typescript
const toggleDay = useCallback(async (dayIndex: number) => {
  const newWorkouts = [...data.workouts];
  newWorkouts[dayIndex] = !newWorkouts[dayIndex];

  // Calculate XP delta BEFORE updating state
  if (options.onXPDelta && options.currentStreak !== undefined) {
    const oldCount = data.workouts.filter(Boolean).length;
    const newCount = newWorkouts.filter(Boolean).length;
    const oldXP = calculateWeeklyXP(oldCount, options.currentStreak, data.status);
    const newXP = calculateWeeklyXP(newCount, options.currentStreak, data.status);
    const delta = newXP - oldXP;
    // Fire callback (can be negative if removing workout)
    options.onXPDelta(delta);
  }

  const newData: WeekData = {
    ...data,
    workouts: newWorkouts,
    updatedAt: new Date(),
  };

  setData(newData);
  await saveData(newData);
}, [data, saveData, options]);
```

**4. Modify `setStatus` function:**
When status changes, XP for the entire week may change (e.g., sick → 0 XP). Call `onXPRecalculate` to trigger full recalculation:

```typescript
const setStatus = useCallback(async (status: WeekStatus) => {
  const newData: WeekData = {
    ...data,
    status,
    updatedAt: new Date(),
  };

  setData(newData);
  await saveData(newData);

  // Status change affects entire week's XP — trigger full recalc
  if (options.onXPRecalculate) {
    options.onXPRecalculate();
  }
}, [data, saveData, options]);
```

**IMPORTANT notes:**
- Do NOT import or depend on useXP directly — callbacks keep this hook independent
- `options` parameter is optional with empty default — existing consumers continue working with no changes
- The XP delta callback fires BEFORE the Firestore write to give optimistic UI update. The actual Firestore XP persistence happens in useXP.addXP.
- Per CONTEXT.md: "XP recalculates immediately on every workout toggle (no debouncing) — formula is pure and fast"
- Per CONTEXT.md: "When toggling workouts on past weeks, use delta-based recalculation"
- The `options` parameter approach avoids breaking existing useWeek consumers (Tracker.tsx, etc.) while adding XP integration capability

**Follow project conventions:**
- `useCallback` with proper dependency arrays
- Preserve all existing functionality unchanged
- No Firebase batch writes in this task (per CONTEXT.md "Claude's Discretion" — keeping existing write pattern for simplicity, atomic batch writes can be added in Phase 7 optimization)

Modify `dismissAchievement` to grant XP after unlock:

```typescript
const dismissAchievement = useCallback(async (achievementId: string) => {
  const achievement = ACHIEVEMENTS.find(a => a.id === achievementId);
  if (achievement) {
    await unlockAchievement(achievement);

    // Grant XP bonus after short delay for dramatic effect (per CONTEXT.md)
    // Achievement toast appears first, then XP increments after a beat
    if (options.onXPGrant) {
      setTimeout(async () => {
        await options.onXPGrant!(100); // +100 XP per achievement, NOT silent (can trigger rank-up)
      }, 800); // 800ms delay per RESEARCH.md recommendation
    }
  }
  setNewlyUnlocked(prev => prev.filter(a => a.id !== achievementId));
}, [unlockAchievement, options]);
```

**Important:** The XP grant uses `setTimeout` with 800ms delay, NOT `await new Promise(resolve => setTimeout(...))`, because the dismiss function should return immediately (don't block the UI). The XP grant fires asynchronously after the delay.

Also update `confirmNewAchievements` similarly:
